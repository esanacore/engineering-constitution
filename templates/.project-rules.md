# Project Engineering Rules

This repository follows [Eric's Engineering Constitution](constitution/CONSTITUTION.md).

## Instructions for AI Agents

As an agent in this workspace, you MUST:
1. **Context Initialization**: Read `constitution/CONSTITUTION.md` and `constitution/AI_WORKFLOW.md`.
2. **Session Planning**: Check `docs/SESSION_PLAN.md` for a previous interrupted session; write your own plan before implementing, and clear or archive it when the session completes.
3. **Project Memory**: Read `docs/MEMORY.md` to load project context, codebase memory, and user preferences. Propose new codebase learnings, user preferences, or major decisions to the user and (upon approval) record them in `docs/MEMORY.md` before completing work.
4. **Project Rules**: Adhere to the conventions in `AGENTS.md`.
5. **Continuous Updates**: Maintain `TODO.md` and `CHANGELOG.md` for every task.
6. **Architecture**: Follow the patterns in `docs/adr/` and `docs/ARCHITECTURE.md`.
7. **Testing**: Never consider a task complete without running or adding automated tests.

If you are using a tool that does not auto-load instructions, please copy this entire file into your system prompt.
