# Testability Refactorer

Use this skill when encountering code that is hard to test (e.g. hidden global state, hardcoded I/O). Refactor to pass dependencies explicitly (Dependency Injection) to allow for easy mocking.

**Constitution Alignment**: This skill strictly enforces the principles laid out in Eric's Engineering Constitution. Always adhere to the established workflows when applying this skill.
