# ADR: Record Architecture Decisions

Status: Accepted

Date: 2026-06-08

## Context

The project needs a consistent way to document major technical decisions.

## Decision

Use Architecture Decision Records in `docs/adr/`.

## Consequences

Important decisions will be easier to review, revisit, and explain to future contributors and AI agents.

## Alternatives Considered

- Keep decisions only in issue comments or pull requests. This was rejected because those records are harder to discover from the repository.
