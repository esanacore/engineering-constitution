# Project Engineering Rules

This repository follows [Eric's Engineering Constitution](constitution/CONSTITUTION.md).

## Critical Workflow

Before performing ANY task, you MUST:
1. Read `constitution/CONSTITUTION.md` (Universal Principles)
2. Read `constitution/AI_WORKFLOW.md` (Required Step-by-Step Workflow)
3. Read `AGENTS.md` (Project-Specific Rules)

## Standards

- Always add or update tests for behavior changes.
- Update `TODO.md` and `CHANGELOG.md` for every task.
- Follow the architectural patterns described in `docs/adr/` and `docs/ARCHITECTURE.md`.
- Review `docs/TROUBLESHOOTING.md` if you encounter errors.
- Record a handoff in `docs/AGENT_HANDOFF.md` before ending your session.
